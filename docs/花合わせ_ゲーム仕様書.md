## 4\. ゲーム進行フロー

### 4.1 局の流れ（概要）

```
局開始
  └── 配札
  └── 交換フェーズ（親→胴二→ビキの順）
  └── 手番ループ（親→胴二→ビキ→親→…）
        └── 手札選択
        └── 場札マッチ・取得
        └── 山札めくり（自動）
        └── 場札マッチ・取得
        └── 次の手番へ
  └── 局終了判定（手札・山札がなくなったら終了）
  └── 精算
  └── 次の局 or ゲーム終了（12局終了後）
```

---

### 4.2 配札

1. 山札をシャッフルし、各プレイヤーに7枚ずつ配る（手札）  
2. 場札に6枚を表向きに並べる  
3. 残り21枚が山札

**場札の特殊処理：**

| 状況 | 処理 |
| :---- | :---- |
| 同月が3枚ある | 3枚を1枚として重ねる。手札・山札からマッチした場合は3枚まとめて取得できる |
| 同月が4枚ある | 全ての場札を山札に戻し、山札をシャッフルして場札を配り直す |

---

### 4.3 交換フェーズ

配札後、親→胴二→ビキの順に交換判定を行う。

**交換条件：** 手札に同月が3枚または4枚ある場合

| 枚数 | 手順 |
| :---- | :---- |
| 3枚 | 該当の手札3枚を伏せる → 手番順の次のプレイヤーが1枚抜き取り山札に戻す → 山札をシャッフル → 山札の一番上1枚を手札に補充 |
| 4枚 | 該当の手札4枚を伏せる → 両隣のプレイヤーが1枚ずつ抜き取り山札に戻す → 山札をシャッフル → 山札の一番上2枚を手札に補充 |

- 交換は任意（しない選択も可）  
- 交換後も同月3枚・4枚の条件を満たす場合は、条件を満たさなくなるまで繰り返し交換できる

---

### 4.4 手番の流れ

#### ① 手札選択

- プレイヤーは手札から1枚を選択して場に出す  
- 場札に同月の札がある場合 → マッチして取得（取り札エリアへ）  
- 場札に同月の札がない場合 → 場に捨てる  
- **場札に合う札があっても、あえて合わない札を捨てることができる**（フケ狙い等）  
- 場札に同月が2枚ある場合 → プレイヤーがどちらを取るか選択できる

#### ② 山札めくり（自動）

- 山札の一番上を1枚めくる  
- 場札に同月の札がある場合 → 自動的に取得  
- 場札に同月の札がない場合 → 場に置く  
- 場札に同月が2枚ある場合 → プレイヤーがどちらを取るか選択できる

CPUの手番における選択（場札2枚マッチ等）はすべて自動で行われる

---

### 4.5 局終了判定

全プレイヤーの手札がなくなった時点で局終了。精算フローへ進む。

## 6\. アニメーション

アニメーションは常時ONとする（OFF設定なし）。

### 6.1 速度設定

設定画面で「速い／普通／遅い」から選択できる。

| 速度 | キー値 |
| :---- | :---- |
| 速い | `fast` |
| 普通 | `normal` |
| 遅い | `slow` |

### 6.2 タイミングと動き

| タイミング | アニメーション |
| :---- | :---- |
| カードを取る | 手札のカードが場札の上に重なり、取り札エリアへフワッと飛んでいく |
| カードを捨てる | 手札のカードが場の空きスペースへスライドして置かれる |
| 山札めくり | 山札の一番上が横にパラっとめくれて場に置かれる |
| 配札 | カードが一瞬で並ぶ（フェードイン） |
| 役成立 | 役名が画面中央に大きくフラッシュ表示されて消える |
| 精算モーダル | 画面中央にフェードインで表示される |
| CPUの手番 | 手札選択の演出なし。取得・捨てのアニメーションのみ適用 |

## 8\. CPU思考ロジック

設定画面で強さを3段階から選択できる。

| 強さ | キー値 | 概要 |
| :---- | :---- | :---- |
| 弱い | `easy` | ランダムに手札を選択する |
| 普通 | `normal` | 役・取り点を考慮した基本的な選択を行う |
| 強い | `hard` | 役・取り点・フケ回避を総合的に考慮した選択を行う |

- CPU1・CPU2は同じ強さ設定を使用する

## 11\. 精算フロー

各局終了後、以下の順序で精算を行う。

| 順序 | 処理内容 |
| :---- | :---- |
| 1 | 取り点計算（光20点・種10点・短冊5点・カス0点） |
| 2 | 柳0点適用：柳を1枚でも取ったプレイヤーは柳の取り点を0点としてフケ判定を行う（フケ不成立の場合は柳の本来点数を遡って加算） |
| 3 | フケ判定 |
| 4 | 雨流れ判定（柳4枚を1人が取得 → 全員役無効） |
| 5 | 役流れ判定（3人全員がフケ以外の役あり → 役無効） |
| 6 | フケ精算 |
| 7 | 役精算（差額精算方式：3ペア間で役点比較、差額を弱い方が支払う） |
| 8 | 最終得点 \= 精算後取り点 − 80点 |

## 15\. UI実装詳細

アニメーションの仕様は「6. アニメーション」、サウンドの仕様は「7. サウンド」を参照。

### 15.1 取り札エリア

- 種別（光・種・短冊・カス）を横並びで表示  
- 4枚以上は12pxオフセットで重ね表示  
- ヘッダー：手番ラベル・プレイヤー名・取り点・役・フケ基準点

### 15.2 タップ操作

| 操作 | 動作 |
| :---- | :---- |
| 手札タップ | 選択状態にする（合う場札をハイライト） |
| ハイライト場札タップ | 選択中の手札を使って場札を取る |
| 捨てるゾーン（🗑）タップ | 手札を捨てる（合う場札なし時のみ有効） |

### 15.3 精算モーダル

- 3列レイアウト（tableレイアウト・上揃え）  
- 列ヘッダー：手番ラベル（👑親 / 胴二 / ビキ）＋プレイヤー名  
- 表示順：取り点 → 役（フケ含む）→ 役流れバナー → 精算内訳 → 最終得点  
- 精算内訳の矢印は相手の方向を向く
