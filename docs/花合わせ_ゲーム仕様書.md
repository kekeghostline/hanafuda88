## 4\. 精算フロー

各局終了後、以下の順序で精算を行う。

| 順序 | 処理内容 |
| :---- | :---- |
| 1 | 取り点計算（光20点・種10点・短冊5点・カス0点） |
| 2 | 柳0点適用：柳を1枚でも取ったプレイヤーは柳の取り点を0点としてフケ判定を行う（フケ不成立の場合は柳の本来点数を遡って加算） |
| 3 | フケ判定 |
| 4 | 雨流れ判定（柳4枚を1人が取得 → 全員役無効） |
| 5 | 役流れ判定（3人全員がフケ以外の役あり → 役無効） |
| 6 | フケ精算 |
| 7 | 役精算（差額精算方式：3ペア間で役点比較、差額を弱い方が支払う） |
| 8 | 最終得点 \= 精算後取り点 − 80点 |

## 8\. UI実装詳細

### 8.1 取り札エリア

- 種別（光・種・短冊・カス）を横並びで表示  
- 4枚以上は12pxオフセットで重ね表示  
- ヘッダー：手番ラベル・プレイヤー名・取り点・役・フケ基準点

### 8.2 タップ操作

| 操作 | 動作 |
| :---- | :---- |
| 手札タップ | 選択状態にする（合う場札をハイライト） |
| ハイライト場札タップ | 選択中の手札を使って場札を取る |
| 捨てるゾーン（🗑）タップ | 手札を捨てる（合う場札なし時のみ有効） |

### 8.3 精算モーダル

- 3列レイアウト（tableレイアウト・上揃え）  
- 列ヘッダー：手番ラベル（👑親 / 胴二 / ビキ）＋プレイヤー名  
- 表示順：取り点 → 役（フケ含む）→ 役流れバナー → 精算内訳 → 最終得点  
- 精算内訳の矢印は相手の方向を向く
